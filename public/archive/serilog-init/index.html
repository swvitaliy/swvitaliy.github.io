Инициализация Serilog

18 июня 2021 г.

Зачастую решение о внедрении того или иного комнонента в проект принимается уже на середине разработки проекта. Так случилось и с компонентом логирования Serilog. Причиной для использования этого компонента было то, что он поддерживает множество провайдеров, а также то, что он поддерживает настройку себя из файла, имеет встроенный лог ротейшн, и поддерживает отправку логов сразу в несколько провайдеров (например, в файл и в консоль).

В проекте есть много строк логирования типа Logger.Log(LogInfo.Information, "line");. Можно конечно их переписать на нативный Serilog логгер, но хотелось бы этого избежать.

Ниже представлено решение, как получить объект класса Microsoft.Extensions.Logging.ILogger, который использовался по умолчанию в прокте, который инкапсулирует в себе объект Serilog.Logger. Причем, настройка логирования происходит из файла конфигурации.

var configBuilder = new ConfigurationBuilder();
Configuration.SetupConfiguration(configBuilder);
var config = configBuilder.Build();

var logger = new LoggerConfiguration()
        .ReadFrom.Configuration(config)
        .CreateLogger();

ILoggerFactory loggerFactory = new LoggerFactory();
loggerFactory.AddSerilog(logger);

var logger = loggerFactory.CreateLogger<Program>();

Метод SetupConfiguration имеет следующий вид:

public static IConfigurationBuilder SetupConfiguration(IConfigurationBuilder builder)
{
    return builder
        .SetBasePath(Directory.GetCurrentDirectory())
        .AddJsonFile("config.json", true)
        .AddJsonFile("local.config.json", true);
}

Данный код можно применять, например, в консольных приложениях, где получить этот объект - это нетривиальная задача).

Пример настроек (файл config.json, в моем случае):

"Serilog": {
  "Using": [ "Serilog.Sinks.Console", "Serilog.Sinks.File" ],
  "MinimumLevel": "Debug",
  "WriteTo": [
    {
      "Name": "File",
      "Args": {
        "path": "logs/log.txt",
        "rollingInterval": "Day",
        "retainedFileCountLimit": 30,
        "outputTemplate": "{Timestamp:yyyy-MM-dd HH:mm:ss.fff} [{Level:u3}] {Message:lj}{NewLine}{Exception}"
      }
    },
    {
      "Name": "Console",
      "Args": {
        "outputTemplate": "{Timestamp:yyyy-MM-dd HH:mm:ss.fff} [{Level:u3}] {Message:lj}{NewLine}{Exception}"
      }
    }
  ],
  "Enrich": ["WithExceptionDetails", "FromLogContext"]
}
