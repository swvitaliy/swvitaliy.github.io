Фильтрация данных в запросах на C#

8 окт. 2021 г.

Есть RPC сервер – несколько обработчиков запросов, которые должны возвращать данные. В один прекрасный момент появилась потребность в фильтрации данных в этих обработчиках. Фильтров много и они появляются в процессе жизни приложения и так же повторно используются в других запросах.

В процессе разработками с фильтрами я решал следующие проблемы

  • Фильтров много – нужно понимать какие фильтры принимает тот или иной запрос и в каком формате
  • Есть несколько запросов, отдающие одинаковые типы данных, для которых нужно применить фильтрацию. Хотелось повторно использовать код
  • Сложная внутренняя структура проекта – хотелось бы передать фильтры из обработчика запроса в модель (в слой работы с данными)

В представленном решении я постараюсь решить эти проблемы.

При получении запроса в обработчике в качестве параметра передается объект с данными запроса. Этот объект запроса может содержать поле с фильтром:

Метод GetQueryFilters возвращает некоторый набор фильтров запроса, тогда как GetQueryFilter возвращает 1 фильтр запроса. Можно  привести оба случая к одному, построив набор вложенных фильтров, обозначающих булеву логику. У меня пока не было такой задачи (применять выражения в фильтрах), поэтому сделал отдельный метод для списка фильтров (они потом через AND конкатэнируются).

В обработчике запроса теперь можно узнать, были ли переданы в запросе какие-то фильтры и вызвать соответствующий метод и, если были переданы, то получить эти фильтры в виде фильтров запросов и передать его в метод модели, работающий с данными.

Класс QueryFilter достаточно тривиален:

public abstract class QueryFilter
{
    public static IQueryable<T> ApplyFilters<T>(IQueryable<T> query, IEnumerable<QueryFilter> filters)
    {
        foreach (var filter in filters)
            query = filter.Apply(query);

        return query;
    }

    public QueryFilterType Kind { get; init; } = QueryFilterType.Unknown;

    public abstract string PrettyValue { get; }

    public abstract IQueryable<T> Apply<T>(IQueryable<T> query);
}

Реализация этого класса выглядит следующим образом:

public class AppointmentByCreatedFilter : QueryFilter
{
    public AppointmentByCreatedFilter()
    {
        Kind = QueryFilterType.AppointmentByCreated;
    }

    public DateTime From { get; set; }
    public DateTime To { get; set; }

    private IQueryable<Entities.Appointment> ApplyFilter(IQueryable<Entities.Appointment> query)
    {
        return query.Where(a => a.Created >= From && a.Created <= To);
    }

    public override string PrettyValue
    {
        get => From.ToString(CultureInfo.InvariantCulture) + " — " + To.ToString(CultureInfo.InvariantCulture);
    }

    public override IQueryable<T> Apply<T>(IQueryable<T> query)
    {
        if (!(query is IQueryable<Entities.Appointment>))
            throw new Exception("Expect appointment query");

        return (IQueryable<T>)ApplyFilter((IQueryable<Entities.Appointment>)query);
    }

Таким образом, при последовательном применении фильтра в цепочке запроса добавляются выражения Where, которые в результирующем SQL запросе конкатенируются через AND.
