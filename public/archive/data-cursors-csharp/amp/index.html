Курсоры данных в C#

swvitaliy - 07 мар. 2021

Есть система, в которой есть разные источники данных. Каждый источник данных может отдавать данные или с пагинацией вида WHERE Id > :prevMaxId или через offset (что не хорошо, но по другому не может). И эти источники данных используют одни и те же алгоритмы.

Чтобы реализовать такую систему я использовал шаблон проектирования курсор данных. Интерфейс курсора данных имеет 2 метода Reset и Next для обхода данных в цикле. Можно было и через IEnumerable, но мне подумалось, что проще сделать так. Если кому-то понадобиться сделать через  IEnumerable – это не составляет труда сделать над ним обертку.

Так же интерфейс IEntityCursor имеет свойство PageCursor, которое возвращает курсор страниц данных. Курсор страниц возвращает постранично данные их БД. Он так же имеет методы Next и Reset для последовательного получения уже страниц данных и свойство InternalCursor.

Интерфейс IEntityCursor инкапсулирует в себе логику работы IEntityPageCursor. Когда в IEntityCursor вызывается метод Next, то возвращается следующий элемент из загруженной страницы данных. Если же страница данных ещё не загружена или в ней не осталось элементов для выдачи и (для обоих случаев) не стоит флаг окончания, то происходит вызов метода Next курсора страниц. Если курсор страниц что-то вернул, то итерируемся по этой новой странице. Иначе, устанавливаем флаг останова и возвращаем false.

Последнее свойство InternalCursor имеет интерфейс IInternalPageCursor. Это интерфейс для внутреннего представления курсора страниц. Он нужен, чтобы устанавливать и получать текущее состояние курсора. Таким образом, в запросе можно указывать параметры смещения (пагинации) в наборе данных. Кроме того, при длительных процессах обработки данных вы можете сохранять курсор время от времени и, в случае нарушения работы и завершения процесса с ошибкой, при повторном запуске подгрузить курсор и продолжить с места, где возникла ошибка (соответствующим образом обработав её).

Метод AssignTo нужен как раз для того, чтобы загружать курсор из БД.

В классах, описывающих запросы к RPC серверу содержится метод, который возвращает соответствующий курсор в зависимости от данных, указанных в запрос.
