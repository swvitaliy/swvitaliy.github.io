Выражения в C#

9 июня 2021 г.

Не так давно столкнулся с проблемой формирования выражений на C#. Если коротко – это такой механизм, который на практике позволяет формировать SQL запросы, остовываясь на C# коде. В теории это действительно хорошая идея, потому что она позволяет писать гомогенный код. Но на практике это совершенная дичь – писать таким образом выражения крайне сложно (явно недопиленная технология).

Зачем мне понадобилось писать "сырые" выражения?

Я написал синхронизацию данных из гетерогенных источников в sqlite. На первом этапе данные были приведены к единому интерфейсу Data Provider, который предоставлял возможность получения данных. Этот интерфейс используется веб сервером. Таким образом, в зависимости от настроек веб сервера при инициализации он получал тот или иной инстанс дата провайдера и, соответственно, работал с нужной БД.

Затем, этот же интерфейс реализует и сам кеш. Поэтому он может использоваться в веб сервере. Причем, данные в кеше имеют ревизию и при запуске процесса синхронизации они должны обращаться к данным только своей ревизии.

Для того, чтобы данные из разных ревизий не попали в ответ от веб сервера используется другое условие, а именно, в каждой сущности есть флаг IsActive. Он установлен для текущей активной ревизии (для новой, в которую идет запись из процесса синхронизации он не установлен). Затем, после успешного завершения процесса синзронизации, этот флаг для новых данных устанавливается, а для старых убирается.

В зависимости от контекста запуска кода, обращающегося к БД, он должен менять условие в запросе на получение данных. Точнее часть условия. Таким образом, есть 2 предусловия:

  • IsActive == true, в случае получения данных текущей ревизии
  • RevNumber == rev_number, в случае процесса синзронизации

Был написан следующий метод для добавления к условию запроса предустовия:

public static Expression<Func<TSource, bool>> PrepareCondition<TSource>(Expression<Func<TSource, bool>> predicate)
{
    if (Mode == ActualQueryableMode.Nothing)
        return predicate;

    if (Mode == ActualQueryableMode.ByRevisionNumber && RevisionNumber == ZeroRevNumber)
        throw new Exception("expect to setup RevNumber");

    Expression<Func<TSource, bool>> fn = Mode switch
    {
        ActualQueryableMode.ByIsActive => (TSource source) =>
            ((IActualSource) source).IsActive,
        ActualQueryableMode.ByRevisionNumber => (TSource source) =>
            ((IActualSource) source).RevisionNumber == RevisionNumber,
        ActualQueryableMode.Nothing =>
            throw new Exception($"unexpected ActualQueryableMode \"{Mode}\""),
        _ => throw new Exception($"unknown ActualQueryableMode \"{Mode}\"")
    };

    var body = Expression.AndAlso(fn.Body, predicate.Body);
    return Expression.Lambda<Func<TSource, bool>>(body, fn.Parameters[0]);
}

В коде используется интерфейс IActualSource для обращения к свойствам RevisionNumber и IsActive. Сущности должны реализовать этот интерфейс.

Данный метод используется для создания методов-расширений:

public static TSource FirstActual<TSource>(this IQueryable<TSource> source,
    Expression<Func<TSource, bool>> predicate) =>
    source.First(PrepareCondition(predicate));

public static TSource FirstOrDefaultActual<TSource>(this IQueryable<TSource> source,
    Expression<Func<TSource, bool>> predicate) =>
    source.FirstOrDefault(PrepareCondition(predicate));

public static IQueryable<TSource> WhereActual<TSource>(this IQueryable<TSource> source,
    Expression<Func<TSource, bool>> predicate) =>
    source.Where(PrepareCondition(predicate));

В результате получил такую ошибку:

System.InvalidOperationException : The LINQ expression 'DbSet<AppointmentResult>()
    .Where(a => ((IActualSource)a).IsActive && ar.Business.Id == __businessInfoEntity_Id_0 && ar.SourceAppointmentResultId == __arm_Id_1)' could not be translated. Either rewrite the query in a form that can be translated, or switch to client evaluation explicitly by inserting a call to 'AsEnumerable', 'AsAsyncEnumerable', 'ToList', or 'ToListAsync'. See https://go.microsoft.com/fwlink/?linkid=2101038 for more information.

Суть сообщения об ошибке – или исправь выражение (как?) или напиши запрос на клиенте. Ну, если бы я мог написать на клиенте, то не мучался бы с написанием выражений. Остается первый вариант – исправить выражение.

При беглом взгляде на сформированное выражение видно, что имя аргумента не совпадает с тем, которое было у начального условия:

a => ((IActualSource)a).IsActive && ar.Business.Id == __businessInfoEntity_Id_0 && ar.SourceAppointmentResultId == __arm_Id_1

Для того, чтобы решить эту проблему необходимо передать в предусловие параметр из передаваемого на вход предиката и в результирующую часть так же передать этот параметр. Кроме того, даже в этом случае выражение не собралось.

В результате я явно постоил выражения предусловия, передав туда параметр и только тогда это заработало:

public static Expression<Func<TSource, bool>> PrepareCondition<TSource>(Expression<Func<TSource, bool>> predicate)
{
    if (Mode == ActualQueryableMode.Nothing)
        return predicate;

    if (Mode == ActualQueryableMode.ByRevisionNumber && RevisionNumber == ZeroRevNumber)
        throw new Exception("expect to setup RevNumber");

    Expression<Func<TSource, bool>> fn;
    switch (Mode)
    {
        case ActualQueryableMode.ByIsActive:
            var param = predicate.Parameters[0];
            var prop = Expression.Property(param, "IsActive");
            var constTrue = Expression.Constant(true);
            var equal = Expression.Equal(prop, constTrue);
            fn = Expression.Lambda<Func<TSource, bool>>(equal, param);
            break;
        case ActualQueryableMode.ByRevisionNumber:
            var param1 = predicate.Parameters[0];
            #pragma warning disable CA1507
            var prop1 = Expression.Property(param1, "RevisionNumber");
            #pragma warning restore CA1507
            var constRevNumber = Expression.Constant(RevisionNumber);
            var equal1 = Expression.Equal(prop1, constRevNumber);
            fn = Expression.Lambda<Func<TSource, bool>>(equal1, param1);
            break;
        case ActualQueryableMode.Nothing:
        default:
            throw new Exception($"wrong ActualQueryableMode \"{Mode}\"");
    }

    var body = Expression.AndAlso(fn.Body, predicate.Body);
    return Expression.Lambda<Func<TSource, bool>>(body, predicate.Parameters[0]);
}

Причем, что характерно, описанный в интернетах способ создания параметра в данном случае не работает

var param = Expression.Parameter(typeof(TSource), predicate.Parameters[0].Name);

Видимо, при формировании SQL из выражения происходит сравнение по ссылке или учитываются какие-то дополнительные поля, отличные от Name.
